# 美术资产 · 视觉资产提示词生成工作流 V1.1

skill作者（抖音）：桐景谯
创建日期：2026-06-11

> 作者信息仅为文档署名，不参与本 skill 的任何规则判断、执行流程或输出内容生成。

> V1.1 更新：① 新增生成角色提示词前的「人物年龄确认」强制流程（短剧出镜年龄≠剧本推理年龄）；② 把「面部锚点（全状态固定）」定为角色资产固定字段；③ 新增「角色造型按需增补」能力（按用户简述追加新造型，五官与确认年龄保持一致）。

**定位**：从剧本中提取角色、场景、道具等视觉资产，生成纯美术资产文生图提示词。  
**接口目标**：为视频提示词生成 skill 提供统一的 `@资产名` 引用，但本 skill 不承担视频分镜、空间调度、连续性约束或镜头执行说明。

## 最高优先级规则

1. 本 skill 只负责视觉资产文生图提示词，不生成视频分镜提示词。
2. 在处理剧本前，必须先向用户确认视觉风格；除非用户已经明确给出风格。
2-A. 在生成任何角色资产提示词前，必须先逐一向用户确认剧本中所有人物的实际年龄。短剧的选角/出镜年龄常与剧本设定或按剧情推理出来的年龄不一致，严禁直接采用剧本字面或推理年龄；一律以用户确认的年龄为准。用户未确认年龄前，不得生成角色资产提示词。
2-B. 每个会出现多状态、或后续可能增补造型的角色，资产组内必须固定写明一行「面部锚点（全状态固定）」，锁定该角色的脸型、骨相、五官结构、核心视觉记忆点与年龄基准。该角色的所有状态/造型提示词都必须显式继承此锚点。
2-C. 当用户要求为某一已存在角色新增造型（换装、特殊场景造型、新身份、新状态等）时，依据用户的简述，把该造型作为一个新的 `#### @角色名_状态词` 单独追加到该角色资产组中，并同步把新 `@角色名_状态词` 补进「可引用 @资产清单」。该新造型必须完整沿用此角色既有的「面部锚点」，保持脸型、骨相、五官、核心视觉记忆点与确认年龄完全一致，只改变服装、妆发、配饰等造型变量；不得借增补造型之机改动五官或另起一个新角色。
3. 角色资产和场景资产文生图提示词固定使用 `21:9`；道具资产文生图提示词固定使用 `16:9`。
4. 资产库只输出 `@资产名`、资产类型、文生图提示词。
5. 默认不输出短描述、连续性约束、空间结构、使用规则、视频交接说明。
6. 角色提示词只写人物本体，最终文生图提示词使用“纯白无缝背景，仅呈现角色本体、服装与随身饰品”等正向约束，不写入场景、背景、职业道具、手边物、车辆、锅具、屏幕、蒸汽等外部对象。
7. 所有场景提示词必须使用“无人空镜场景，仅呈现空间结构、固定陈设和环境质感”等正向约束；自检时确认没有人物、人脸、路人或主体角色。
8. 门内人、电话声、OS 声音、回忆中未露面的对象，若剧本没有可见形象，不得脑补正面人物视觉。
9. `参考风格基调` 只用于场景资产提示词；角色资产和道具资产不得直接写入“雨夜、港片雨夜、霓虹街景、冷暖街灯、老城区场景”等会污染背景或环境的风格词。
10. 不是所有资产都生成多状态；必须由剧本中的可见变化、时间变化、物理变化、信息变化或明确回忆段落触发。
11. 只有当状态变化会改变静态资产图的视觉识别，并且后续镜头需要稳定复用时，才生成多状态。
12. 多状态资产必须保留同一资产的基础一致性锚点，只改变剧本中明确发生变化的状态变量。
13. 状态变化只影响视频动作、表演节奏或镜头调度时，不生成新的资产状态。
14. 单一角色默认 1-4 个状态，单一场景默认 1-5 个状态，单一道具默认 1-4 个状态；超过上限时合并相近状态。
15. 面向 GPT 图片模型输出时，最终文生图提示词优先使用正向目标，不大量使用“不要、禁止、无、非”等负向描述。
16. 最终文生图提示词避免使用会诱发过锐或堆料的词，如“精细、复杂、繁复、浓郁、HDR、高细节、8K、超高清、锐利、ultra sharp、hyper-detailed、intricate”。
17. 电影胶片风格包只在用户选择“电影写实、真人写实、写实电影感、现实主义影像”等写实类风格时调用；3D、2D、日系、赛博朋克、插画、动画等非写实类风格不默认调用。
18. 外部参考资料中的动作中段、镜头景别、双人对手戏、人物在场景中做什么等内容属于视频/分镜提示词，不进入本视觉资产库。
19. 固定通用尾缀：每个会实际生成图像的最终文生图提示词（角色、场景、道具，含各自多状态与增补造型）的中文句末之后，必须原样附加固定英文尾缀【Remove the noise and high-frequency details from the image. Keep all the lines, colors, and brightness unchanged.】。该尾缀紧跟在提示词末尾、置于同一文生图提示词代码块内；不可见声音角色的占位说明不生成图像，不加该尾缀。

## Step 0：视觉风格确认

当用户要求“使用本 skill 处理剧本”且未明确给出风格时，必须先询问用户，不得直接生成资产库。

询问时必须先提醒：

```text
建议使用 GPT Image 2 生成本资产库中的图片提示词；本 skill 已按 GPT 图片模型的出图习惯优化提示词结构。
```

必须确认：

```text
1. 视觉风格：写实电影感 / 赛博朋克 / 3D / 2D / 日系 / 其他
2. 参考风格基调：例如《隐秘的角落》冷现实主义 / 霓虹赛博老城区 / 港片雨夜 / 日式清冷生活流 / 其他
```

可选确认：

```text
1. 角色设定图是否需要三视图。
2. 场景图是否需要空镜版本。
3. 道具图是否需要白底设定图。
```

如果用户回复“按默认来”，使用：

```text
视觉风格：真人写实电影感
参考风格基调：现实主义悬疑短片，低饱和冷暖对比
角色设定图：右侧为角色头部高清特写，左侧为角色全身三视图（正面/侧面/背面）
场景图：空镜，无人物
道具图：中性背景或白底高清设定图
```

## Step 0.5：人物年龄确认（强制，先于一切角色提示词）

完成风格确认后、生成任何角色资产提示词之前，必须先做人物年龄确认。原因：短剧的选角与出镜年龄常与剧本设定年龄、或按剧情推理出来的年龄不一致（例如剧本写主角 16 岁，实际出镜可能是 18-25 岁的演员）。若直接采用剧本字面或推理年龄，会导致出图年龄严重失真。

执行方式：

1. 先通读剧本，列出所有需要资产化的人物（主要角色 + 有台词/关键表演的配角 + 一次性角色），并标注每个人物在剧本中的字面/推理年龄，仅作为参考。
2. 以清单形式逐一向用户确认每个人物的**出镜年龄**，请用户给出准确数字或区间。可附上参考年龄供用户快速校正。
3. 对存在年龄跨度的角色（幼年/少年/壮年/老年/回忆等多状态），需分别确认每个年龄阶段的出镜年龄。
4. 用户确认前，不得生成任何角色资产提示词；用户确认后，角色提示词中的年龄一律以用户给定值为准，不再回退到剧本推理年龄。

确认清单示例格式：

```text
请确认以下人物的出镜年龄（左为剧本参考年龄，请给出实际出镜年龄）：
- 主角A：剧本约16岁 → 出镜年龄？
- 反派B：剧本约16岁 → 出镜年龄？
- 长辈C：剧本约72岁 → 出镜年龄？
- 多状态角色D：幼年(约6岁)/成年(约20岁)/老年(约50岁) → 各阶段出镜年龄？
（可回复"按剧本年龄"则沿用参考年龄）
```

## 资产分类

### 1. 角色资产

适用对象：
- 主要角色。
- 有台词或关键表演的人物。
- 需要跨镜头保持外貌一致的人物。

面部锚点（全状态固定）：每个多状态角色、以及任何后续可能增补造型的角色，在资产组标题下必须先写一行面部锚点，固定锁定脸型、骨相、五官结构、核心视觉记忆点与确认年龄。该角色的每个状态/造型提示词都显式继承此锚点，只改变服装妆造等变量，确保换装或增补新造型时同一张脸始终一致。

角色提示词只允许包含：
1. 年龄、身份、职业气质、地域气质。
2. 脸型、骨相、五官结构。
3. 1-2 个核心视觉记忆点。
4. 眼睛、眉毛、鼻子、嘴唇、发型。
5. 皮肤质感、生活痕迹。
6. 服装、饰品。
7. 情绪气质，但不写夸张表演。

角色提示词禁止包含：
- 场景、背景、街道、店铺、房间、楼道。
- 职业道具、手边物、车辆、外卖箱、锅具、屏幕、伞、手电。
- 光源环境、蒸汽、雨景、门缝、桌椅等外部对象。
- “雨夜、港片雨夜、霓虹街景、街灯、老城区街道”等场景氛围词。
- 具体镜头、机位、运镜、动作段。

### 2. 场景资产

适用对象：
- 故事发生的主要空间。
- 跨镜头重复出现的位置。
- 与美术风格、空间气氛有关的场景局部。

场景提示词必须包含：
1. 时间、天气、地点。
2. 空间结构和可见动线。
3. 光源位置和色温。
4. 场景中的固定物和关键陈设。
5. 场景质感、年代感、生活痕迹。
6. 固定视觉符号。
7. 使用正向约束表达为无人空镜场景，仅呈现空间结构、固定陈设和环境质感；自检确认没有人物、人脸、路人或主体角色。

### 3. 道具资产

适用对象：
- 剧情关键道具。
- 角色反复触碰或观看的物件。
- 需要近景保持一致的物件。

道具提示词必须包含：
1. 尺寸和比例参照。
2. 材质、磨损、使用痕迹。
3. 剧情识别点。
4. 中性背景或白底设定图。

道具提示词禁止包含：
- 角色正在使用道具的动作。
- 完整场景、复杂背景、多人互动。
- “雨夜、港片雨夜、霓虹街景、街灯、老城区街道”等场景氛围词，除非道具本身必须表现雨水、泥点、磨损等物理痕迹。
- 视频镜头、机位、运镜。

## 资产多状态判断规则

### 1. 多状态总原则

先判断资产是否必须资产化，再判断是否需要状态化。多状态不是扩写装饰，而是为了解决同一角色、场景、道具在剧本中出现明显静态视觉差异时的稳定复用问题。

必须生成多状态的情况：
- 同一资产在剧本中出现明显可见差异。
- 该差异会影响文生图结果的外观识别。
- 该差异后续可能被视频提示词多次引用。

不生成多状态的情况：
- 只是角色动作、表情瞬间、站位、坐姿、转身、奔跑。
- 只是镜头远近、机位、运镜、剪辑节奏。
- 只是短暂情绪变化，且不改变服装、妆发、伤痕、年龄、体态等静态外观。
- 只是场景中一次性光影变化，且不影响场景固定美术设定。

视觉差异不足的状态必须合并，例如“疲惫、失眠、虚弱、崩溃”如果外观差别不大，可合并为一个 `@角色名_疲惫` 状态。

### 2. 角色多状态触发

角色出现以下变化时，才生成多状态：
- 年龄阶段变化，如幼年、成年、老年。
- 换装或身份伪装，如校服、婚服、制服、伪装身份。
- 受伤、病态、血污、烧伤、明显疲惫等静态外观变化。
- 发型、妆容、体态、整洁程度发生明显变化。
- 现实与回忆中的人物形象不同。
- 时间跨度造成外貌变化。

角色多状态必须保留：
- 脸型、骨相、眉眼、鼻嘴结构、核心发型特征。
- 1-2 个核心视觉记忆点。
- 基础身份气质。

角色多状态只能改变：
- 年龄、服装、伤痕、污损、妆发、体态、整洁程度、长期情绪气质。

角色状态名可以来自剧情，但提示词不得写入场景。例如 `@艾莉_受伤` 可以写湿发、衣物污损、脸部擦伤，不得写雨夜街道、门口、车辆、灯光环境。

### 3. 场景多状态触发

同一场景出现以下变化时，才生成多状态：
- 白天、夜晚、黄昏、深夜等时间造成明显美术差异。
- 正常、破败、灾后、焚毁、废弃等物理状态变化。
- 现实、回忆、梦境等段落造成明确视觉状态变化。
- 整洁、凌乱、被搜查、被破坏、长期无人等陈设状态变化。
- 天气或光源变化成为该场景的关键视觉识别点。

场景多状态必须保留：
- 空间结构、可见动线、固定陈设、材质年代感、核心视觉符号。

场景多状态只能改变：
- 时间、天气、光源、损坏程度、陈设状态、清洁程度、色温、气氛。

场景资产可以写“正面全景设定图、俯视结构设定图、空镜设定图”等构图视角，但不得写具体镜头运动、分镜执行或摄影机调度。

### 4. 道具多状态触发

道具出现以下变化时，才生成多状态：
- 完整、破损、沾血、烧毁、泥污、老化等物理状态变化。
- 合上、打开、半开、拆开、封存等开合状态变化。
- 空白页、关键页、署名页、报警界面、裂痕照片等信息状态变化。
- 被包裹、露出一角、完整显露等隐藏状态变化。
- 剧情关键标记、裂痕、编号、文字、贴纸、污渍发生变化。

道具多状态必须保留：
- 尺寸比例、材质、形状、颜色、基础磨损、核心识别符号。

道具多状态只能改变：
- 开合、损坏、污染、文字内容、显露程度、关键标记。

道具状态不得写人物手持动作、使用过程、完整场景或视频镜头。

### 5. 状态命名规则

多状态资产命名使用：

```text
@资产名_状态词
```

规则：
- `@资产名` 是资产组名，默认不单独作为生成图提示词。
- `@资产名_状态词` 是可生成、可引用的具体状态资产。
- 状态词 2-6 个汉字优先，避免过长。
- 状态词只写视觉状态，不写完整句子。

推荐：

```text
@艾莉_成年
@艾莉_幼年
@艾莉_受伤
@温室仓_正常
@温室仓_破败
@温室仓_回忆
@日记本_合上
@日记本_关键页
@日记本_烧毁
```

禁止：

```text
@艾莉_雨夜街头崩溃大哭
@温室仓_深夜红灯故障破败状态
@日记本_被男主拿在手里翻到关键页
```

## GPT 图片模型适配规则

### 1. 正向约束优先

GPT 图片模型对负向提示词不稳定，写“不要 X”可能反而激活 X。因此最终文生图提示词优先写成正向目标：

```text
纯白无缝背景，仅呈现角色本体、服装与随身饰品
无人空镜场景，仅呈现空间结构、固定陈设和环境质感
白底或中性背景，仅呈现单一道具
```

内部规则和自检可以继续判断“不能出现什么”，但最终输出给模型的提示词应尽量少写“禁止出现、不要、无、非”等负向句式。

### 2. 避免过锐和堆料

最终文生图提示词避免使用：

```text
精细、复杂、繁复、浓郁、HDR、高细节、8K、超高清、锐利、ultra sharp、hyper-detailed、intricate、HDR photography
```

推荐替换为：


## 角色造型按需增补

资产库生成后，用户可随时要求为某个已存在角色追加新造型（换装、特殊场景造型、新身份伪装、受伤/华服等新状态）。此为增量操作，按以下流程执行：

1. 定位该角色已有的资产组与「面部锚点（全状态固定）」。若该角色此前是单状态、没有写锚点，先据其既有提示词提炼出面部锚点并补写，再继续。
2. 依据用户的简述（往往只有一两句，如“给主角加一套红色喜服造型”），把简述扩写成符合本 skill 字段规范的造型描述：仅写服装、妆发、配饰等造型变量，保持角色提示词的纯净（不写场景、背景、手边物、镜头等）。
3. 以 `#### @角色名_状态词` 形式，把新造型作为一个独立状态追加到该角色资产组末尾。状态词取 2-6 个汉字、只写视觉状态。
4. 新造型提示词开头显式继承面部锚点（“同一面部锚点的……”），并沿用用户已确认的出镜年龄；只改变造型变量，五官、骨相、核心视觉记忆点保持完全一致。
5. 比例沿用角色资产的 `21:9`，并套用与该角色同款的 GPT 质感尾缀与风格设置。
6. 把新增的 `@角色名_状态词` 同步补入「可引用 @资产清单」。
7. 仅追加，不改动该角色其它已存在的状态；除非用户另有要求。

增补示例：

```text
用户：给主角加一套大婚红色喜服的造型。
追加：
#### @主角名_喜服
文生图提示词：
（比例 21:9 + 同款风格设置 + 同一面部锚点的脸型/骨相/五官/记忆点 + 用户确认年龄；
 造型改为：大婚红色喜服、簪冠/喜帕、相应妆发；其余保持一致 + 同款质感尾缀。）
```

## 输出流程

### Step 1：提取视觉资产

内部建立三类清单，默认不输出完整分析表：

| 类型 | 原文名称 | 建议资产名 | 是否必须资产化 | 是否需要多状态 | 状态依据 | 建议状态 |
|:---|:---|:---|:---:|:---:|:---|:---|

判断标准：
- 跨镜头出现：必须资产化。
- 影响剧情：必须资产化。
- 影响美术风格：必须资产化。
- 只是背景装饰：可不资产化。
- 有剧本证据且静态视觉识别明显变化：需要多状态。
- 只有动作、表情、机位或剪辑变化：不需要多状态。

每个资产只输出：
- `### @资产名`
- `类型：角色 / 场景 / 道具 / 不可见声音角色`
- `文生图提示词`

如果资产需要多状态：
- `### @资产名` 只作为资产组标题。
- 每个状态使用 `#### @资产名_状态词` 输出。
- 每个状态都必须有独立文生图提示词。
- 不输出状态分析表，除非用户明确要求。

- 连续性约束。
- 空间结构。

输出所有可被视频 skill 使用的 `@资产名`。
